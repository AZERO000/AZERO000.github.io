常量引用

引用

  1. 引用是对象的别名，她本身不是对象，只是已经存在对象的另一个名字；所有对引用的操作均是对它所代表的对象的操作。

  2. 初始化：
    定义一个引用的时候，必须初始化，其初始化值必须是一个对象，引用一旦初始化完成，将始终和他的初始值对象绑在一起

     例如： int a=1;
       int &ra=a;
       ra++;  //现在ra是a的别名，对ra的操作，其实就是对a的操作
 错例：（1）int &ra=10;   //初始化值必须是一个对象
      （2）double d=3.14;
          int &rd=d;     //类型不匹配

常量引用

  1. 初始化：与前边普通引用的初始化不同，常量引用允许用任意表达式作为其初始值，只要该表达式的结果可以转换成引用的类型即可。 其实，在上边三个例子中，初始化为不可寻址的值或者类型不匹配的对象，编译器这时候会生成临时对象，并将常量引用绑定到该临时变量上 因此，普通引用中，绑定对象必须严格的匹配，若类型不匹配，则定义的引用其实会绑定到临时对象中，而对引用进行操作时，实际上操作的是临时对象
  • const引用仅对可进行的操作做了限定，引用的对象本生没有限定

常量指针和指针常量

常量指针

常量指针：本质上是一个指针，常量来修饰它，表示这个指针是一个指向常量的指针，而指针本身是一个变量。（指针指向的对象是常量）

  • 注意，const只是修饰词，只是说明不能通过指针来修改，并不是指针指向的一定是常量，例如：1. int strcmp(const char * ,const char *)

指针常量

指针是形容词，常量是一个名词；本质上是一个常量，用指针来修饰它，即这个常量是指针中存储的值

这个指针变量中存储的地址不能改变。

指向常量的指针常量

本质：是一个常量，这个常量是一个指针（即指针中存储的地址不变），且通过地址寻址的值也是常量。
